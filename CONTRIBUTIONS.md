\# Contributing Guidelines

This document provides guidelines for contributing to the "AI Personalizing and Optimizing Learning Experiences" project code, specifically for group members: Mauricio Gonzalez, Cole Peterson, and Faizan Shaikh.

\#\# Communication

\*   Please discuss any significant changes or new features with the group before starting implementation.  
\*   Use GitHub Issues (if the repository is on GitHub) to track bugs or suggest enhancements. Otherwise, communicate directly via preferred group channels.

\#\# Code Style

\*   Try to follow \[PEP 8\](https://www.python.org/dev/peps/pep-0008/) style guidelines for Python code.  
\*   Add comments (\`\#\`) to explain complex logic.  
\*   Use docstrings (\`"""Docstring goes here"""\`) for functions and classes to explain their purpose, arguments, and return values.

\#\# Submitting Changes (If using Git/GitHub)

1\.  Create a new branch for your changes: \`git checkout \-b feature/your-feature-name\` or \`bugfix/issue-description\`.  
2\.  Make your changes and commit them with clear, descriptive messages.  
3\.  Push your branch to the remote repository: \`git push origin your-branch-name\`.  
4\.  Create a Pull Request on GitHub, describing your changes and linking to any relevant issues.  
5\.  Wait for review and discussion before merging.

\#\# Submitting Changes (If not using Git branches extensively)

\*   Ensure you have the latest version of the code before making changes.  
\*   Discuss your intended changes with the group.  
\*   Clearly communicate when changes are made and pushed/shared.
